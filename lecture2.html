C++ Basics

A typical C++ program will have the following form


        //hello.cpp
        #include<iostream>

        using namespace std;

        int main(){                 //Start of the main function
            cout << "Hello, world"; //First line to be executed
            cout << endl;
            return 0;               //End of the main function
        }

You compile and run the program in the following way


        $ g++ hello.cpp
        $ ./a.out
        "Hello, world"

Comments

Comments are lines that are ignored by the compiler

Comments serve many purposes:

  • Explain to the reader what the program is doing
  • Explain to the reader how pieces of code work
  • A to-do list of tasks to be completed

In C++ there are two ways to write comments

  • inline comments

    inline comments begin with //

    Whatever appears to the right of // is ignored by the compiler

    //For example, this is a comment in C++
  • Multi-line comments

    Multi-line comments begin with /* and end with */

    Whatever is in between /* and */ is ignored by the compiler

    
                /* For example,
                   This is a multi-line comment in C++
                */

Literals and Variables

Both literals and variables have a value and a type. A literal represents a specific value. A variable has a name which refers to a location in memory, and its value may change.

Common types:

Type Description Example
int Integer (positive or negative) 5,-3

Some more notes:

  • string literals are surrounded by double quotes
  • char literals are surrounded by single quotes
  • int, double, and bool literals are not surrounded by quotes
  • "" is an empty string

Declaring Variables

Before you use a variable, you must declare it by specifying its type and name. A variable's type cannot change once specified.

Points to note:

  • A variable name must be a sequence of letters, digits, and underscore (_) characters, and may not begin with a digit.
  • A variable name may not be a reserved keyword
  • You can declare multiple variables of the same type in one statement by using commas (see example below).
  • You cannot declare a variable twice in the same scope (to be defined later), and you cannot access a variable from outside the score in which it was declared.

Examples

The syntax for declaring an int variable named x is:


            int x;

The syntax for declaring an double variable named y is:


            double y;

to declare multiple variables: 


            double x, y, z;

Initializing Variables

Initializing a variable means to assign it an initial value. You must initialize a variable before you try to access its value because it's default value may be garbage

To assign a value to a variable you use the assignment operator =, which is the equal sign.

The following examples are ways in which you can initialize variables: 


            int x = 5;          //Declare x and initialize it to 5 on the same line
            
            double y;
            y = 3.5;            //Declare y and initialize it to 3.5 on separate lines

            char s = 't';       

            int b = x           //Declare b and initialize it to the value of variable x

            int a = 2, c = 3;   //Declare multiple variables and initialize them on the same line

Input

The cin object and the stream extraction operator >> are used to retrieve input from the user

For example, 


            string name;
            cin >> name;

will cause the program to wait until the user types a string of characters and presses enter. Whatever the user typed is assigned to the variable name.

Points to note:

  • whitespace acts as a separator. For example, in the above situation if the user enters "James Kaczmarczyk", only "James" is stored in the variable name. The space " " in between "James" and "Kaczmarczyk" is ignored and "Kaczmarczyk" will remain in the buffer to be used in the next cin statement.
  • cin can be chained just like cout
  • cin uses the stream extraction operator >>
  • cout uses the stream insertion operator <<

Arithmetic

Operators

An expression is a sequence of operators and operands (an operand can be a literal or a variable).

  • An expression is evaluated one operator at a time, in order from highest to lowest precedence.
  • Anything inside parentheses is evaluated before the operators surrounding the parentheses.
  • Operators of the same precedence are evaluated in the direction of their associativity.
  • Arithmetic operators do not modify variables. The assignment operator changes the value of the variable on the left.

Below are some operators from highest to lowest precedence, and their associativity

Precedence Operator Description Associativity
1 *, /, % multiplication, division, modulo left-to-right
2 +, - addition, subtraction left-to-right
3 = assignment right-to-left
